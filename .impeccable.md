## Design Context

### Users
- **Primary**: Recruiters and hiring managers evaluating AI PM candidates — they scan quickly, need to see impact metrics and real shipped products within seconds
- **Secondary**: Potential co-founders and collaborators — looking for someone who can both strategize and build
- **Tertiary**: Industry peers and community — following Charles's writing and building journey

Usage context: Desktop-first browsing (recruiters at desks), but must work flawlessly on mobile (quick shares via LinkedIn/messaging).

### Brand Personality
**Bold. Impactful. Memorable.**

The site should feel like walking into a product demo, not reading a resume. Every section should make visitors think "this person builds real things." Confidence without arrogance — let the work speak, but present it with conviction.

### Aesthetic Direction
- **Tone**: Dark, cinematic, tech-forward — inspired by xAI's visual language
- **Typography**: Space Grotesk (geometric, modern) + SF Mono (technical accents)
- **Palette**: Near-black (#0A0A0A) base, warm mars orange (#E8652B) accent, cyan (#00D9FF) for interactive states
- **Canvas animations**: Interactive particle systems and data visualizations that tell each project's story
- **References**: xAI (layout structure, spacing, section headers), Linear (changelog prose style)
- **Anti-references**: Wix/Squarespace templates (generic, forgettable), plain GitHub README portfolios (no design effort), over-animated award sites (distraction over substance)

### Design Principles
1. **Show, don't tell** — Canvas animations visualize what each product does. Metrics are prominent. The portfolio itself is proof of building ability.
2. **Content density with breathing room** — Pack meaningful information but use generous vertical spacing between sections. No filler, no fluff.
3. **Interactive but purposeful** — Every animation serves a narrative (route tracing for Path, K-line for Plutus Trade, spec assembly for Playbook). No decoration-only effects.
4. **Consistency over novelty** — Unified `[ SECTION ]` headers, consistent spacing tokens, same animation patterns across components. System > individual flourishes.
5. **Performance is a feature** — Canvas animations pause off-screen, IntersectionObserver for reveals, lean dependencies. Speed is part of the brand.
