# AI 痕跡規則庫（掃描用）

改寫與自檢時逐類掃描。每條規則固定三欄：**識別信號 → 等級 → 修法**。
等級定義（詳見 SKILL.md）：

- **L0 鐵律**：命中即修，修不掉就刪，未歸零不得交付。
- **L1 強制**：預設一律修，除非使用者明示保留。
- **L2 建議**：依情境判斷。

使用原則：

1. **模式優先、詞表兜底**：沒列到的新說法，先問它屬於哪一類既有模式，不要求逐詞命中。
2. **禁止換湯不換藥**：刪掉一句套話，不得補上同族的另一句：刪「標誌著」不能補「象徵著」；刪「賦能」不能補「加值」；刪「總的來說」不能補「整體而言」。
3. **刪掉之後不必補**：空話刪掉留下的洞，用具體事實補；沒有事實就讓它空著，或標「（需作者補充：⋯⋯）」。

## 第一類：內容空轉（C1–C9）

### C1 誇大意義與趨勢 — L0
- 信號：標誌著、象徵著、見證了、體現了、彰顯了、突顯了、奠定基礎、開啟新篇章、掀開序幕、轉捩點、里程碑、劃時代、具有重要意義
- 修法：換成可查證的具體事實（時間、數字、名字、事件）；寫不出具體內容就整句刪。
- ❌「這次改版標誌著產品邁入全新里程碑」→ ✅「這次改版把註冊流程從 7 步砍到 3 步」

### C2 過度強調關注度 — L1
- 信號：獲得多家媒體報導、引發熱烈討論、廣受好評、備受矚目、掀起話題（皆無出處）
- 修法：寫得出具體來源就寫（哪家媒體、幾則留言），寫不出就刪。
- ❌「上線後廣受好評」→ ✅「上線一週，有 3 個使用者主動寫信來說找到了想要的功能」

### C3 空洞分析收尾 — L1
- 信號：句尾掛「展現了／體現了／反映出＋抽象名詞」（對用戶體驗的重視、對品質的堅持）
- 修法：砍掉分析尾巴，留下動作本身。
- ❌「我們重寫了錯誤訊息，展現對使用者體驗的重視」→ ✅「我們重寫了錯誤訊息，每一條都告訴使用者下一步能按什麼」

### C4 廣告宣傳腔 — L1
- 信號：充滿啟發、扎實豐富、令人期待、突破自我、全新高度、你所需要的一切、不容錯過、絕佳選擇
- 修法：改成「具體教什麼、解決什麼、給誰用」。
- ❌「一堂充滿啟發的課程」→ ✅「教你寫 prompt、接 API、把 demo 變成能收費的產品」

### C5 模糊歸屬（無源權威） — L0
- 信號：業界專家普遍認為、不少使用者表示、研究顯示、數據表明、被廣泛認為（皆無出處）
- 修法：改成第一手觀察（我看到、我問了誰）、補具體出處，或刪。**絕不代編來源。**
- ❌「研究顯示遠端工作效率更高」→ ✅「我們團隊改遠端後，衝刺完成率從 70% 升到 85%」

### C6 公式化前景段 — L1
- 信號：文末固定一段「儘管⋯⋯但⋯⋯」「迎接充滿挑戰的未來」「持續努力、不斷精進」
- 修法：換成具體的下一步和時間點，或直接刪掉整段。
- ❌「未來仍充滿挑戰，我們會持續努力」→ ✅「下一步是把回覆延遲壓到一秒內，目標九月上線」

### C7 均值回歸（可遷移性違規） — L0
- 信號：整段讀起來都對、都正向，但把主詞換成任何同類產品／公司／人也完全成立；專屬細節為零。
- 測試法：隨機抽一段，把主詞換掉，還成立就是命中。
- 修法：塞回只有這個主題才有的細節（專屬數字、事件、名字），塞不回就刪。

### C8 幻覺引用 — L0（標記，不代查、不代刪）
- 信號：精確到小數點卻查無此研究、張冠李戴的名人語錄、點不開的連結、無頁碼的引用
- 修法：原句保留，句前加「〔需查證來源〕」，交回作者處理。不刪、不改寫、不幫忙補一個新來源。

### C9 立場真空 — L1
- 信號：各有優缺點、因人而異、見仁見智、取決於多方面因素、需視具體情況而定
- 修法：整段沒有判斷就是沒有內容。補上作者的實際選擇與理由；作者沒給就標「（需作者補充：你自己選哪個、為什麼）」，不代編。
- ❌「兩種框架各有優缺點，取決於團隊需求」→ ✅「我選 Vue，因為團隊三個人裡兩個只會 Vue，招聘現實比技術品味重要」

## 第二類：句構與語言（S1–S13）

### S1 系詞迴避 — L1
- 信號：作為⋯⋯、扮演著⋯⋯的角色、服務於、定位為、致力於成為
- 修法：用「是」直說。
- ❌「作為團隊的核心產品之一」→ ✅「是今年的主力產品」

### S2 排比三段式 — L1
- 信號：形容詞、形容詞、形容詞（無縫、直觀、強大）或短語×3，每項都沒有事實支撐
- 修法：拆開，只留寫得出內容的那一項。
- ❌「更快、更穩、更安全」→ ✅「查詢從 3 秒降到 0.4 秒」

### S3 否定平行結構 — L0（全篇限額 1 次）
- 信號：不是 A 而是 B、不只是 A 更是 B、不僅⋯⋯更⋯⋯、與其說是 A 不如說是 B
- 修法：全篇最多一次，其餘改直述句、時間結構或轉折結構。
- ❌「這不僅是工具，更是一種思維方式」→ ✅「它是查資料的工具，用久了會順便改掉你亂存檔案的習慣」

### S4 刻意換詞循環 — L1
- 信號：同一個人／物在同段落內換多種代稱（創作者⋯⋯內容工作者⋯⋯夥伴）
- 修法：固定用同一個名字或代名詞，重複沒有罪。

### S5 虛假範圍 — L1
- 信號：「從 X 到 Y」但兩端沒有實質關聯或涵蓋性（從入門到精通、從理念到實踐）
- 修法：改成真實的配對，或一個具體主張。
- ❌「從概念到落地的完整旅程」→ ✅「先做假門測需求，兩週後上第一版收費頁」

### S6 解說導引句 — L0
- 信號：這意味著、換句話說、也就是說（濫用）、值得注意的是、值得一提的是、需要指出的是、更重要的是、把⋯⋯擺在一起看、這背後透露的訊息是
- 修法：刪掉導航墊片，直接講結論。「這意味著」要過三關：誰在推論？根據什麼？結論對誰有效？答不出就整句刪。

### S7 勸誡反問收尾 — L1
- 信號：與其⋯⋯不如⋯⋯＋格言＋「不是嗎？」、只有⋯⋯才能⋯⋯、讓我們拭目以待
- 修法：換成作者的具體經驗；沒有經驗就刪。
- ❌「與其焦慮，不如行動，不是嗎？」→ ✅「我把每天回信的事交給範本，多出來的 40 分鐘拿去寫作」

### S8 說教式深度腔 — L1
- 信號：說到底、歸根究柢、本質上、究其根本、真正的問題在於、核心在於、關鍵不在⋯⋯而在⋯⋯
- 修法：刪掉儀式化前綴，直接說那個洞見本身。

### S9 金句／格言公式 — L0（全篇限額 1 次）
- 信號：X 是 Y 的 Z（筆記是思考的鏡子）、X 是新的 Y（數據是新的石油）、這是一場⋯⋯的革命、真正的 X 是 Y
- 修法：還原成具體主張。全篇最多留一句真的有內容的。

### S10 假坦白鉤子 — L1
- 信號：說真的、老實說、講白了、不騙你、跟你說個秘密（作為獨立開場）
- 修法：刪掉報備式停頓，讓陳述自己站著。

### S11 戲劇性短句轟炸 — L1
- 信號：連續三句以上極短句／殘句堆疊，句句想砸出史詩感（「然後一切都變了。沒有預兆。沒有回頭路。」）
- 修法：併成一到兩句有主詞、有因果的句子，保留一個短句當重音就好。

### S12 公式化開場（時代大帽子） — L0
- 信號：在當今瞬息萬變的時代、在數位轉型的浪潮下、隨著 AI 技術快速發展、在資訊爆炸的社會、近年來⋯⋯蔚為風潮
- 修法：整段刪掉。第一句就要有只有這篇文章才有的資訊：一個場景、一個動作、一個數字、一個具體問題。

### S13 四字格與對仗堆疊 — L1
- 信號：連續四字成語鋪排（深耕細作、穩紮穩打、行穩致遠）、上下句刻意對仗、駢文腔
- 修法：留一個最準的，其餘改白話直述。台灣媒體文風是白話為主，成語當調味不當主菜。

## 第三類：風格與版面（F1–F9）

### F1 半形標點混用 — L0
- 信號：中文句子裡出現半形 `,` `.` `:` `;` `?` `!` 或英文引號 `""`
- 修法：一律轉全形「，。：；？！」；引號用「」『』。例外：英文詞組、網址、程式碼。

### F2 破折號 — L0（內文零容忍）
- 信號：內文出現任何破折號（——、—、──），一次就算違規
- 修法：改逗號、句號、冒號或括號。破折號是 AI 文最顯眼的格式指紋，中文永遠有替代標點，歸零沒有損失。
- 例外：標題的主副標分隔「──」（台灣媒體慣例，見 tw-media-style.md）；引號內的原話不改。

### F3 粗體過度 — L0（≤2 處／段）
- 信號：一句話內多處加粗、整段粗體密度高
- 修法：每段最多 2 處；社群等純文字平台 0 處（`**星號**`會直接露出）。

### F4 內聯粗體標籤列表 — L1
- 信號：「**標籤**：說明」格式縱向重複（**效能**：提升。**體驗**：改善。）
- 修法：改回散文段落。真的需要列點時，每點要是完整的資訊句。

### F5 編號切碎段落 — L0
- 信號：首先／其次／再者／最後把文章等分；把散文硬拆成「1. 抽象名詞：定義」
- 修法：恢復散文；結構要服從邏輯，不是服從對稱。硬湊的那一點刪掉，過渡詞多半可直接拿掉。

### F6 Emoji 堆疊 — L0（預設 0，社群情境 ≤1）
- 信號：一則內容多個 emoji、emoji 當列點符號（🚀💡✅🔥）
- 修法：對外文章、Email、文書一律 0 個；社群貼文最多 1 個。

### F7 表格誤用 — L1
- 信號：一句話講得完的資訊硬做成表格；「優點／缺點／意義」三欄全是抽象詞
- 修法：改回散文。表格只留給真正的多維比較（3 個以上方案 × 價格 × 限制那種）。

### F8 句長過勻 — L1
- 信號：連續三句以上長度、結構幾乎相同
- 修法：打斷一句變超短句，或併兩句變長句。最重要的內容用最短的句子。

### F9 標題 AI 腔 — L1
- 信號：一文看懂、全面解析、深度解讀、終極指南、必看、乾貨滿滿、「XXX：YYY」冒號公式連發
- 例外：「一次看懂／一表看懂／懶人包」是台灣媒體服務型報導的正當用法，內容真的是整理包（附表格或多點拆解）才可用，詳見 tw-media-style.md 仲裁區。
- 修法：照 tw-media-style.md 的標題公式重下：給數字、給懸念、給具體利益或具體人物，不給「本文很完整」的自我宣告。

## 第四類：溝通殘留（R1–R7）

### R1 協作交流殘留 — L0
- 信號：希望這對你有幫助、如果需要調整請告訴我、以下是修改後的版本、可直接複製使用、祝你順利
- 修法：整句刪除。這些是聊天室話語，不屬於文章。

### R2 諂媚語氣 — L0
- 信號：好問題！、完全正確、很棒的觀察、你問到核心了
- 修法：刪掉頒獎前言，直接回應內容。

### R3 通用積極結論 — L0
- 信號：總的來說、綜上所述、總而言之、整體而言（收尾用）、充滿無限可能、邁向美好未來、讓我們一起、共同努力、如果你也有同感，歡迎留言告訴我（罐頭互動）
- 修法：換成具體下一步，或**允許不收尾**，文章可以停在最後一個具體句子上。

### R4 過度避險（hedging 疊加） — L1（≤1 層）
- 信號：可能、或許、潛在地、一定程度上、某種程度上，兩層以上疊加
- 修法：確定的事 0 層避險；真不確定的事最多 1 層，並講清楚不確定在哪。
- ❌「可能會在一定程度上產生潛在影響」→ ✅「會拖慢首屏載入，實測多 0.3 秒」

### R5 知識截止免責 — L0
- 信號：由於資訊有限、無法確認最新情況、根據現有資料、作為一個 AI
- 修法：查證後直說；沒查就用人話坦白：「我沒查最新版，以官網為準。」

### R6 模板佔位未填 — L0（標記，不代填）
- 信號：（此處填入品牌名）、[產品名稱]、XX 公司、[在這裡插入案例]
- 修法：逐一標出，請作者填。不代編。

### R7 預告式導言 — L0
- 信號：廢話不多說、話不多說直接進入、讓我們一起來看看、接下來帶你了解、先說結論（但後面沒馬上說）、以下一一拆解
- 修法：刪掉預告，直接從第一個有內容的句子開始。

## 第五類：工具殘留（T1–T2）

### T1 AI 工具殘留物 — L0
- 信號：`utm_source=chatgpt.com`、`utm_source=openai`、`referrer=grok.com`、`turn0search0`、`citeturn0news0` 等私有碼
- 修法：清掉參數與殘碼，保留連結本體。

### T2 Markdown 標記錯置 — L1
- 信號：`**星號**`、`# 井號`、`- 列表符` 出現在不支援 Markdown 的平台（FB、IG、Threads、LINE）
- 修法：發布前轉換：粗體改「引號」或口語強調，標題改換行，列表改逐行短句。
